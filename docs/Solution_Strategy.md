# Solution Strategy — «Клёво»

> SPARC Phase 2 · SOLVE · First Principles + TRIZ + Game Theory  
> Дата: 2026-04-09

---

## Problem Statement (SCQA)

- **Situation:** Российская молодёжь 18–28 лет тратит больше, чем зарабатывает. Безналичных платежей 83% — данные о тратах цифровые и доступны. BNPL-сервисы выросли на 300%+ за 2023–2024.
- **Complication:** Существующие инструменты (банковские приложения Сбера, Т-Банка) скучны и не мотивируют. Финансовая тревожность у 67% молодёжи — но осознание не превращается в действие.
- **Question:** Как создать финансовый инструмент, который Gen Z будет использовать не из-за обязанности, а с удовольствием — и который вирально распространяется сам?
- **Answer:** AI-ассистент с характером («Жёсткий режим»), встроенный в Telegram, с геймификацией и вирусным механизмом шеринга roast'а.

---

## First Principles Analysis

### Фундаментальные истины

1. **Деньги — это данные.** При 83% безналичности все транзакции уже оцифрованы. Барьер — не отсутствие данных, а их фрагментация по банкам.

2. **Развлечение > Скука.** Люди используют то, что приносит удовольствие или социальное признание. Финансовые приложения проигрывают TikTok за внимание — но «roast» — это контент, которым хочется поделиться.

3. **Стыд блокирует, юмор разблокирует.** Финансовая тревожность создаёт избегание. Смех снижает психологический барьер к действию.

4. **Telegram — нативная среда ЦА.** 82% аудитории 16–24 в Telegram ежедневно. Zero-friction установка через TMA.

5. **Вирусность = нулевой CAC.** Каждый пользователь, делящийся roast'ом, привлекает следующего бесплатно.

### Выводы из First Principles

- Продукт должен быть **развлечением с финансовым побочным эффектом**, а не финансовым инструментом с развлекательным интерфейсом
- Данные нужно получать с минимальным трением (CSV, ручной ввод — не Open Banking на MVP)
- Вирусность — это архитектурная фича, а не маркетинг

## Root Cause Analysis (5 Whys)

**Проблема:** Молодёжь не управляет финансами

1. **Почему?** → Не знают, куда уходят деньги
2. **Почему?** → Данные о тратах разбросаны по 2–3 банкам и BNPL-сервисам
3. **Почему?** → Нет единого агрегатора, адаптированного под российские реалии
4. **Почему?** → Западные инструменты (Mint, Cleo) не работают в РФ; российские банки не мотивируют на анализ
5. **Почему?** → **Root Cause:** Финансовое поведение молодёжи не ассоциируется с удовольствием или социальным статусом в РФ-контексте

**Решение root cause:** Создать положительный социальный контекст вокруг финансового самосознания через юмор, шеринг и gamification.

## Game Theory Analysis

### Ключевые игроки

| Игрок | Интересы | Стратегия | Угроза |
|---|---|---|---|
| Пользователь | Контроль трат, развлечение | Попробует бесплатно, заплатит за «больше» | Уйдёт если скучно |
| Т-Банк / Сбер | Удержание клиентов, cross-sell | Могут скопировать roast-механику | Высокая |
| ЮKassa | Процессинг платежей | Комиссия с каждой подписки | Смена тарифов |
| Telegram | Рост mini apps экосистемы | Продвигает TMA в каталоге | Политика изменится |
| Регулятор (ЦБ РФ) | Защита потребителей | Нейтрален к информационным сервисам | Изменение трактовки |

### Nash Equilibrium

Стабильная стратегия: **Freemium с сильной бесплатной версией** — пользователь получает ценность бесплатно → делится → новые пользователи → рост, который защищает от копирования крупными игроками (им нужны месяцы на реализацию, мы растём быстрее).

### Кооперативная стратегия

Т-Банк и Сбер — потенциальные **партнёры** (Open API для данных), а не только конкуренты. Партнёрство выгодно им: мы улучшаем financial health их клиентов → меньше дефолтов.

## Second-Order Effects

| Первый порядок | Второй порядок | Третий порядок |
|---|---|---|
| Пользователи видят свои реальные траты | Начинают сокращать ненужные расходы | Изменяется поведение рынка BNPL-сервисов |
| Roast становится вирусным | Нормализация «говорить о деньгах» среди Gen Z | Рост финансовой грамотности как культурный тренд |
| Продукт собирает данные о тратах Gen Z | Уникальный датасет о паттернах потребления 18–28 | Потенциал B2B (аналитика для ретейла) |
| Успех в РФ | Возможность экспансии в СНГ (Казахстан, Беларусь) | Региональный финтех-лидер |

## Contradictions Resolved (TRIZ)

| Противоречие | TRIZ-принцип | Решение |
|---|---|---|
| Нужна честность (критика) ↔ не нужно обидеть пользователя | **Принцип 3: Местное качество** | Roast настраивается по интенсивности; «Мягкий режим» и «Жёсткий режим» |
| Нужны данные для анализа ↔ пользователи не хотят делиться данными | **Принцип 10: Предварительное действие** | Демо без данных → aha-moment → добровольная загрузка данных |
| Нужна монетизация ↔ барьер для подписки | **Принцип 25: Самообслуживание** | Пользователь сам «закрывает» себе базовые функции → просит больше |
| Нужно быть уникальным ↔ банки могут скопировать | **Принцип 27: Дешёвая недолговечность** | Бренд и характер нельзя скопировать быстро; технология вторична |
| Нужны данные из нескольких банков ↔ нет Open Banking в РФ | **Принцип 15: Динамичность** | MVP: CSV-импорт; v1: Т-Банк API; v2: универсальный агрегатор |

## Recommended Approach

### Стратегия: «Entertainment-First Fintech»

**Ядро:** Продукт позиционируется и воспринимается как **развлечение** (Roast Mode, мемы, характер), а финансовая польза — как «побочный эффект».

**Три слоя ценности:**

```
Слой 1 — Развлечение (БЕСПЛАТНО):
  → «Жёсткий режим» · Roast на основе трат
  → Вирусный шеринг скриншотов
  → Aha-moment: «вот куда уходят деньги»

Слой 2 — Инструмент (₽199/МЕС):
  → Автосбережения
  → Финансовые цели
  → История без ограничений
  → Умные уведомления

Слой 3 — Платформа (ROADMAP):
  → Open API агрегация
  → Цифровой рубль
  → B2B аналитика
```

### Ключевые стратегические ставки

1. **Ставка на характер, не на функции** — бренд «Клёво» с узнаваемым tone-of-voice труднее скопировать, чем CSV-импорт
2. **Вирусность как продуктовая архитектура** — каждый экран содержит шеринг-механику
3. **Данные в РФ как конкурентное преимущество** — ФЗ-152-compliance с первого дня, западные игроки не могут войти
4. **Telegram-first** — нулевые расходы на дистрибуцию через органику каталога TMA
