# Solution Strategy — Стрики и ачивки

> SPARC Phase 2 · SOLVE  
> Дата: 2026-04-14

---

## Problem Statement (SCQA)

- **Situation:** Клёво имеет 8 завершённых MVP-фич, хорошую функциональность (CSV импорт, роаст, цели)
- **Complication:** Пользователи не возвращаются регулярно — нет habit loop. Одноразовое использование убивает retention
- **Question:** Как создать механику, заставляющую Gen Z возвращаться в Клёво каждый день/неделю?
- **Answer:** Streak + Achievement система с Telegram-нативным шерингом

---

## First Principles Analysis

1. **Люди возвращаются в приложения, которые дают им мгновенное вознаграждение.** Streak = visual reward за повторение
2. **Социальное давление в Telegram > любого push-уведомления.** Шеринг ачивок создаёт FOMO у друзей
3. **Ачивка ценна только если сложно получить.** Нельзя давать всё сразу — нужна прогрессия
4. **Простота реализации важнее полноты.** 2 типа стриков + 9 ачивок достаточно для v1

---

## Root Cause Analysis (5 Whys)

1. Почему retention низкий? → Нет причины возвращаться
2. Почему нет причины возвращаться? → Приложение решает задачу за одну сессию
3. Почему нет повторяющейся задачи? → CSV импорт — редкое событие
4. Почему редкое? → Пользователь не видит ценности в частом отслеживании
5. **Root Cause:** Нет механики, связывающей ежедневное финансовое поведение с немедленной наградой в приложении

---

## Game Theory Analysis

| Игрок | Интересы | Конфликт | Решение |
|-------|----------|----------|---------|
| Пользователь FREE | Бесплатные фичи, мотивация | Не хочет платить | Все ачивки бесплатны — стрик стимулирует возврат |
| Пользователь PLUS | Ценность за деньги | Должен видеть разницу | История стриков только PLUS |
| Клёво | MAU, конверсия | Не хочет перегрузить фичами | Минимум UI, максимум автоматизма |
| Telegram | Рост платформы | — | Шеринг через native API — win-win |

**Nash Equilibrium:** Все ачивки доступны FREE → пользователи активны → MAU растёт → конверсия в PLUS через другие PLUS-фичи

---

## Second-Order Effects

1. Streak создаёт FOMO при потере → grace period снижает отток
2. Шеринг ачивок → органический рост → новые пользователи → новые стрики (вирусный loop)
3. Ачивка `BUDGET_MASTER` (4 недели без overspend) → пользователи начинают реально следить за тратами → больше транзакций → лучше AI-данные → лучший роаст

---

## Contradictions Resolved (TRIZ)

| Противоречие | TRIZ Принцип | Решение |
|--------------|--------------|---------|
| Геймификация мотивирует, но создаёт иллюзию | Принцип 40 (Composite materials) | Ачивки за реальные действия (импорт, цели), а не за нажатие кнопок |
| Стрик нужно сбросить (честность) vs. не сбрасывать (retention) | Принцип 15 (Dynamics) | Grace period 1 день — компромисс между честностью и retention |
| Шеринг → виральность vs. конфиденциальность | Принцип 10 (Prior action) | Шеринговая карточка не содержит суммы, только название ачивки |

## Recommended Approach

### 1. Two-Streak Model
- **Import Streak**: +1 за каждый день с новым CSV-импортом или ручным обновлением цели. Grace period: 1 день (не сбрасывается если пропустил 1 день)
- **Spending Streak**: +1 за каждую неделю с тратами < предыдущей недели. Вычисляется каждое воскресенье в 23:59 по транзакциям

### 2. Event-Driven Achievements
- `AchievementService.checkAndUnlock(userId, event)` вызывается из каждого route-хэндлера после ключевого действия
- Идемпотентно: `@@unique([userId, achievement])` в БД — повторные вызовы безопасны
- 9 ачивок, unlock = once-per-lifetime

### 3. Telegram Share
- `Telegram.WebApp.shareMessage()` или `switchInlineQuery` → пользователь шарит карточку ачивки
- Карточка: эмодзи + название + "Клёво помогает мне с деньгами 🔥"

### 4. Minimal DB Footprint
- `UserStreak` (1 row per user) + `UserAchievement` (up to 9 rows per user)
- Streak read → Redis cache → DB fallback

## Risk Assessment

| Риск | Вероятность | Impact | Mitigation |
|------|-------------|--------|------------|
| Пользователи накручивают стрик | Низкая | Низкий | Импорт CSV — реальное действие |
| Grace period слишком мягкий | Средняя | Средний | Аналитика через 30 дней → настройка |
| Ачивки воспринимаются как детские | Средняя | Средний | Дизайн с характером (в стиле роаст-режима) |
| Spending streak ломается при отсутствии данных | Высокая | Низкий | No data this week → streak freeze (не +1, не -1) |
