# PRD: Шеринг и виральность (viral-sharing)

## Overview

Viral-sharing — механизм, превращающий каждый сгенерированный Roast в точку вирусного роста. Пользователь делится брендированной карточкой своего «жёсткого режима» в Telegram, друг видит, переходит по реферальной ссылке, регистрируется. K-factor цель: > 0.3.

## Problem Statement

Текущий `onShare` просто открывает `openTelegramLink` — голая ссылка без контекста. Нет визуального крючка, нет реферального трекинга, нет CTA для получателя. Конверсия практически нулевая.

## Target Users

- **Шарящий:** 18-28 лет, только что получил smeshной roast, хочет показать друзьям
- **Получатель:** видит пост от друга в Telegram, любопытствует «что это за приложение»

## Business Value

- Viral coefficient K > 0.3 → органический рост без затрат на маркетинг
- Referral program → дополнительный engagement (пользователь следит за конверсиями)
- «Поделиться ростом» → повышает retention (возвращается смотреть статистику)

## Key Features (MVP)

### 1. Share Modal с брендированной карточкой
- Открывается при нажатии «Поделиться» после roast
- Показывает визуальную карточку: топ-кат траты + цитата из roast + лого Клёво
- Кнопка «Отправить другу» — Telegram native share
- Кнопка «Скопировать ссылку» — clipboard

### 2. Реферальная ссылка и трекинг
- У каждого пользователя есть уникальный `referralCode` (уже в схеме)
- Share URL включает referralCode: `https://t.me/bot?start=ref_<code>`
- GET /referral — возвращает мой код, число приглашённых, число активных

### 3. Реферальная статистика в профиле
- Мини-секция в настройках/профиле: «Приглашено: N человек»
- Визуальный стимул: «Пригласи 3 друга — получи бонусный roast»

## Out of Scope (MVP)

- Генерация PNG/OG-изображений сервером (требует puppeteer/satori)
- Push-уведомление при регистрации реферала
- Монетизация рефералов (cashback, скидка) — следующая итерация
- Leaderboard рефералов

## Success Metrics

| Метрика | Цель |
|---------|------|
| % roast сессий с кликом «Поделиться» | > 15% |
| Конверсия share → новый пользователь (K) | > 0.3 |
| Referral stats page views/MAU | > 20% |

## Constraints

- Telegram WebApp: нет прямого API генерации Stories (только `shareToStory` в новых версиях)
- Дизайн карточки — HTML/CSS в TMA, не изображение (MVP ограничение)
- `referralCode` и `referredBy` уже в схеме User — новых DB-сущностей нет
- Все роуты защищены JWT
