# PRD: Dashboard анализа трат — Клёво

## Overview

Dashboard — центральный экран приложения Клёво. Показывает пользователю анализ его трат с разбивкой по категориям, сравнением периодов и навигацией к другим фичам.

## Problem Statement

После импорта CSV пользователь видит «чёрный ящик»: деньги как бы ушли, но непонятно куда и сколько. Dashboard даёт немедленный ответ на вопрос «куда ушли деньги?»

## Target Users

Gen Z, 18–28 лет, Россия. Только что загрузил выписку из Сбера/Т-Банка. Хочет быстро понять свои траты без таблиц Excel.

## Core Value Proposition

«За 3 секунды вижу, куда ушли деньги в этом месяце» — с разбивкой по категориям и сравнением с прошлым месяцем.

## Key Features (MVP)

### F1: Period Selector
- Переключатель: Этот месяц / Прошлый месяц / 3 месяца
- Дашборд пересчитывается при смене периода

### F2: Spending Summary
- Общая сумма трат за период
- Сравнение с предыдущим периодом (+/-% и абсолютная дельта)

### F3: Category Breakdown
- Топ-5 категорий (список + процент + сумма)
- Pie chart (Recharts) для визуализации
- RU-специфичные иконки и названия категорий

### F4: Analytics API
- `GET /analytics/summary?period=month|last_month|3months`
- Серверная агрегация (не клиентская) для производительности
- Кэширование Redis 5 минут

## Non-Functional Requirements

- API response: ≤200ms (с Redis кэшем)
- Нет внешних LLM вызовов — только DB aggregation
- Все суммы в копейках (integer)

## Success Metrics

- 70%+ пользователей открывают дашборд после импорта
- Bounce rate Dashboard < 30%
- Time-to-insight < 3 сек (от открытия до понимания трат)
