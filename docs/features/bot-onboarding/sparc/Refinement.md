# Refinement: Bot Onboarding

## Edge Cases

| Сценарий | Ввод | Ожидание | Обработка |
|----------|------|----------|-----------|
| Повторный /start | Существующий юзер | Онбординг снова | Показать, не создавать дубль |
| Устаревший callback | Нажатие на старую кнопку | Ошибка grammY | try/catch + answerCallbackQuery с текстом |
| Пустое ctx.from | Бот или анонимный чат | Нет имени | Дефолт "друг" |
| Длинное имя | first_name > 50 символов | Некрасивый текст | Обрезать до 20 символов |
| Реферальный + bank выбор | /start ref_XXX нажал Сбер | Оба обработаны | Сохранить referral, показать онбординг |

## Testing Strategy

### Unit Tests
- `sendWelcome` — проверить текст и структуру keyboard
- `sendSberInstructions` — проверить все 6 шагов присутствуют
- `sendTbankInstructions` — проверить все 6 шагов присутствуют
- `handleBankSelect` — проверить routing по callback data

### Integration Tests
- `/start` → получить сообщение с 3 кнопками банков
- Нажать "Сбербанк" → получить инструкцию + кнопку TMA
- `/how` → получить выбор банка
- `/banks` → получить список банков

## Security

- Callback data не содержит user ID или других секретов
- `ctx.from?.id` всегда проверяется на null
- Реферальный код валидируется через БД, не напрямую
