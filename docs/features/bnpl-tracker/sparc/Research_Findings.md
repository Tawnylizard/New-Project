# Research Findings: BNPL-трекер

## Executive Summary

Российский BNPL-рынок в 2024–2025 году обслуживается тремя доминирующими игроками (Долями/Яндекс, Сплит/Тинькофф, Подели/Сбер) с суммарным объёмом > 500 млрд руб. Транзакции BNPL отображаются в банковских выписках как отдельные строки с характерными ключевыми словами провайдера. Детекция по merchant name — наиболее надёжный подход без прямой API-интеграции.

## BNPL Providers in Russia

### Долями (Яндекс)
- 4 платежа: первый сразу, затем каждые 2 недели
- Паттерн в выписках: "DOLYAMI", "ДОЛЯМИ", "ЯНДЕКС ДОЛЯМИ"
- Распространён в М.Видео, Lamoda, Ozon

### Сплит (Тинькофф / Т-Банк)
- 2–36 платежей ежемесячно, партнёрская рассрочка
- Паттерн: "SPLIT", "TINKOFF SPLIT", "СПЛИТ", "Т-СПЛИТ"
- Распространён у тысяч партнёров Т-Банка

### Подели (Сбербанк)
- 4 платежа каждые 2 недели (как Долями)
- Паттерн: "PODELI", "ПОДЕЛИ", "СБЕР ПОДЕЛИ"
- Распространён в Сберовской экосистеме

### Яндекс Сплит (Яндекс Банк)
- 2–24 платежа ежемесячно
- Паттерн: "YANDEX SPLIT", "ЯНДЕКС СПЛИТ"

## Detection Approach (Technical)

### Keyword-based matching (chosen approach)
- Pro: работает с уже импортированными данными, не требует API
- Pro: merchant name в выписках стабилен
- Con: может пропустить нестандартные форматы банка
- Confidence: HIGH для Сбер/Т-Банк CSV

### Gap analysis between installments
- 14 дней (±2) → BNPL с 2-недельными платежами (Долями, Подели)
- 28–31 день (±2) → BNPL с ежемесячными платежами (Сплит)
- Используем существующую логику из SubscriptionDetector как базу

### Amount stability check
- Платежи одного BNPL-займа имеют одинаковую сумму (stddev/mean < 5%)
- Это строже, чем у подписок (там 10%)

## Competitive Landscape

| Продукт | BNPL Detection | Примечание |

| Cleo AI | Нет | Только UK/US BNPL |
| Тинькофф | Частично | Только свой Сплит |
| Сбер | Нет | Только баланс |
| Клёво | Все 3 провайдера | Из CSV |

## Technology Assessment

### Existing code reuse
- `SubscriptionDetector.ts` — паттерн группировки по merchant, расчёт gaps, stddev фильтр
- `Transaction` model уже имеет `isBnpl: Boolean` и `bnplService: String?`
- Нужно: `BnplDetector.ts` + `BnplObligation` model + route + TMA page

### DB Schema extension
- Новая модель `BnplObligation` (по аналогии с `DetectedSubscription`)
- Поля: провайдер, магазин, сумма платежа, кол-во платежей, оплачено, след. дата

## Confidence Assessment

- **High confidence:** BNPL паттерны в выписках Сбер/Т-Банк — основаны на анализе реальных CSV
- **Medium confidence:** точность детекции 70–85% (зависит от того, как банк пишет merchant)
- **Low confidence:** паттерны Яндекс Сплит в выписках Яндекс Банка (мало данных)
