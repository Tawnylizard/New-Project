# PRD: BNPL-трекер (bnpl-tracker)

## Overview

BNPL-трекер — функция Клёво для автоматического обнаружения и агрегации обязательств по рассрочке («купи сейчас, плати потом») из транзакций пользователя. Цель: дать Gen Z ясную картину долговой нагрузки от сервисов Долями, Сплит и Подели.

## Problem Statement

Российские BNPL-сервисы (Долями/Яндекс, Сплит/Тинькофф, Подели/Сбер) стали повсеместны, но разрознены: пользователь одновременно платит частями в нескольких сервисах и теряет контроль над суммой долга. Нет единого места, где видна вся картина.

## Target Users

- Основные: 18–28 лет, активные пользователи Долями/Сплит, работающие или студенты
- Боль: «Не помню, сколько ещё платить за наушники из М.Видео»
- Контекст: Используют Клёво после CSV-импорта выписки из Сбера или Т-Банка

## Business Value

- Повышает retention: пользователь возвращается в приложение проверить следующий платёж
- Дифференциация: ни один конкурент в RU-рынке не агрегирует BNPL из выписок автоматически
- Upsell: «Клёво Плюс» включает уведомления о BNPL-платежах за 3 дня

## Key Features (MVP)

### 1. Сканирование транзакций на BNPL
- Автоматическое распознавание провайдера по ключевым словам в merchant name
- Поддержка: Долями, Сплит (Тинькофф), Подели (Сбер), Яндекс Сплит
- Группировка платежей одного BNPL-займа

### 2. Агрегация обязательств
- Подсчёт оплаченных и оставшихся платежей
- Расчёт следующей даты платежа (проекция)
- Суммарный долг по всем активным BNPL

### 3. Timeline платежей
- Хронологическая лента: когда и сколько платить
- Визуальное разделение: просрочено / на этой неделе / будущие
- Индикатор прогресса (3/4 оплачено)

### 4. Детальная карточка BNPL
- Имя магазина (Zara, М.Видео, etc.)
- Провайдер (Долями/Сплит/Подели)
- Статус: active / completed / overdue

## Out of Scope (MVP)

- Push-уведомления (только для Плюс, следующая итерация)
- Ручное добавление BNPL (только авто-сканирование из CSV)
- Интеграция с API Долями/Сплит/Подели напрямую
- Dispute/спор по транзакции

## Success Metrics

| Метрика | Цель | Период |
|---------|------|--------|
| % пользователей с ≥1 BNPL обнаруженным | > 30% | 30 дней после релиза |
| Avg сессий на пользователя с BNPL | > 2.5 / неделю | |
| BNPL scan % от общих сканирований | > 60% | |

## Constraints

- Данные только из уже импортированных CSV-транзакций
- ФЗ-152: данные только в Yandex Cloud ru-central1
- Суммы всегда в копейках (integer), никогда float
- Все маршруты защищены JWT-аутентификацией
