# Specification: BNPL-трекер

## User Stories

### US-1: Сканирование BNPL
```
As a пользователь Клёво с импортированной выпиской,
I want to запустить сканирование BNPL-платежей,
So that я увижу все свои активные рассрочки в одном месте.

Acceptance Criteria:
Given пользователь импортировал CSV с транзакциями
When он нажимает "Найти рассрочки" на странице BNPL
Then API сканирует транзакции, обнаруживает BNPL-паттерны
And возвращает список BnplObligation с провайдером, магазином и суммами
And в транзакциях проставляется isBnpl=true для найденных платежей
```

### US-2: Просмотр обязательств
```
As a пользователь с активными BNPL,
I want to видеть все рассрочки со статусом и прогрессом,
So that я знаю, сколько ещё платежей осталось.

Acceptance Criteria:
Given есть обнаруженные BnplObligation
When пользователь открывает страницу BNPL
Then он видит карточки: провайдер, магазин, сумма платежа, прогресс X/N
And виден следующий платёж и его дата
And есть суммарная строка: «Итого долг: X ₽»
```

### US-3: Timeline платежей
```
As a пользователь с несколькими рассрочками,
I want to видеть хронологический список будущих платежей,
So that я могу спланировать бюджет.

Acceptance Criteria:
Given 2+ активных BnplObligation
When пользователь открывает вкладку Timeline
Then платежи отсортированы по дате (ближайшие сверху)
And просроченные выделены красным
And каждая запись показывает: дата, сумма, провайдер, магазин
```

### US-4: Управление обязательством
```
As a пользователь,
I want to скрыть ошибочно найденное BNPL-обязательство,
So that оно не засоряет мой трекер.

Acceptance Criteria:
Given открыта карточка BnplObligation
When пользователь свайпает влево и выбирает "Скрыть"
Then статус меняется на dismissed
And обязательство исчезает из основного списка
And totalDebt пересчитывается без этого обязательства
```

## Non-Functional Requirements

### Performance
- Сканирование ≤ 1000 транзакций: < 200 мс
- GET /bnpl: < 100 мс (из DB, без пересчёта)

### Security
- Все роуты требуют JWT (requireAuth)
- Prisma queries всегда scope userId
- Zod schema на каждый роут

### Data Integrity
- Суммы в копейках (integer), никогда float
- Upsert на `@@unique([userId, bnplService, merchantName, startDate])`
- onDelete: Cascade при удалении User
