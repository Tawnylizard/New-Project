# Solution Strategy — Финансовые цели

> SPARC Phase 2 · SOLVE  
> Дата: 2026-04-14

---

## Problem Statement (SCQA)

- **Situation:** Пользователи Клёво уже импортируют CSV и видят свои траты
- **Complication:** Нет инструмента для постановки финансовой цели и понимания, как её достичь с текущим уровнем трат
- **Question:** Как добавить goals-функцию, которая использует существующие транзакции для AI-рекомендаций и монетизируется через PLUS?
- **Answer:** CRUD-модуль целей с ручным прогрессом + AI-анализ трат → конкретный план экономии, доступный только PLUS-подписчикам

---

## First Principles Analysis

1. Финансовая цель — это дельта между текущим состоянием и желаемым, ограниченная временем
2. AI полезен тогда, когда у него есть данные (транзакции уже есть в БД)
3. Монетизация через paywall работает, если FREE-версия достаточно полезна, чтобы создать habit, но недостаточна для полной ценности
4. Деньги всегда в копейках — никакого float-арифметики

---

## Root Cause Analysis (почему пользователи не достигают целей)

1. Why? → Нет конкретной цели → нет мотивации откладывать
2. Why? → Нет визуального прогресса → не видно, куда движется
3. Why? → Нет связи трат с целью → непонятно, что срезать
4. Why? → Данные разбросаны по разным банкам → нет единой картины
5. **Root Cause:** Нет единого инструмента, который объединяет цель, прогресс и анализ реальных трат

---

## Game Theory Analysis

| Игрок | Интерес | Стратегия |
|-------|---------|-----------|
| FREE-пользователь | Бесплатные инструменты | 1 цель без AI — достаточно для habit formation |
| PLUS-пользователь | Максимальная ценность | Unlimited goals + AI советы = retention lock-in |
| Клёво | Конверсия FREE→PLUS | Paywall после создания 1 цели → триггер апгрейда |

**Nash Equilibrium:** FREE-пользователь создаёт цель, видит paywall на AI, апгрейдится → все выигрывают.

---

## Contradictions Resolved (TRIZ)

| Противоречие | TRIZ Принцип | Решение |
|-------------|-------------|---------|
| AI-советы ценны, но нельзя давать бесплатно | #35 Parameter Change | Дать структуру (прогресс-бар) бесплатно, AI — платно |
| Транзакции содержат PII, нельзя в LLM | #10 Preliminary Action | Агрегировать по категориям перед отправкой |
| Дедлайн может не существовать | #15 Dynamism | Дедлайн опционален; без него AI даёт советы без прогноза |

---

## Recommended Approach

### Backend
1. Новая Prisma-модель `FinancialGoal` (userId, name, targetAmountKopecks, currentAmountKopecks, deadline?, category, status, aiAdvice?)
2. CRUD routes: `GET/POST /goals`, `PUT/DELETE /goals/:id`
3. AI route: `POST /goals/:id/advice` — PLUS-only, генерирует совет через YandexGPT → GigaChat → cached fallback
4. GoalService отвечает за бизнес-логику и LLM-вызовы

### Frontend
1. Страница `Goals.tsx` — список целей + кнопка создания
2. `GoalCard.tsx` — прогресс-бар, статус, кнопка AI-совета (с PLUS-paywall)
3. `CreateGoalModal.tsx` — форма создания цели
4. `UpdateProgressModal.tsx` — обновление текущей суммы

### Монетизация
- FREE: 1 активная цель, нет AI-советов → paywall на кнопке «Получить AI-совет»
- PLUS: Unlimited goals, AI-советы без ограничений

## Risk Assessment

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| YandexGPT timeout | Средняя | Высокое | GigaChat fallback → cached advice |
| Пользователь забывает обновлять прогресс | Высокая | Среднее | Напоминания через бота (v2) |
| AI генерирует нерелевантный совет | Средняя | Среднее | Retry до 2 раз, content validation |
| FREE пользователь разочарован 1-goal лимитом | Средняя | Среднее | Чёткое объяснение в UI + легкий апгрейд |
