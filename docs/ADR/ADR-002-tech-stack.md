# ADR-002: Технологический стек

**Дата:** 2026-04-09  
**Статус:** Принято  
**Децидер:** Tech Lead

---

## Контекст

Нужно выбрать стек для MVP российского AI-финансового ассистента с учётом:
1. Санкционных ограничений (OpenAI недоступен напрямую без VPN)
2. Требований хранения данных на территории РФ (ФЗ-152)
3. Telegram как основного канала дистрибуции
4. Команды из 2–4 разработчиков на старте

---

## Рассмотренные варианты

### Frontend

#### Вариант А: Telegram Mini App (TWA) + React (выбранный)
- Telegram Web Apps SDK + React + Vite
- Встроен в Telegram, нет необходимости в App Store
- Используется TailwindCSS для быстрой вёрстки
- **За:** нулевой CAC для установки, 82% ЦА уже в Telegram
- **Против:** ограниченный доступ к нативным API устройства

#### Вариант Б: Next.js Web App + PWA
- Классический веб с PWA-оберткой
- Хостинг на российском облаке (Yandex Cloud / Selectel)
- **За:** полный контроль над UX, SEO
- **Против:** высокий CAC (нужен органический трафик), медленнее установка

#### Вариант В: React Native (iOS + Android)
- **За:** нативный UX, доступ к уведомлениям
- **Против:** долгий ревью AppStore/Google Play, высокая стоимость разработки

---

### Backend

#### Вариант А: Node.js + Fastify (выбранный)
- TypeScript, монорепо с фронтом
- **За:** единый язык в команде, быстрый старт, большая экосистема
- **Против:** не лучший выбор для CPU-heavy задач

#### Вариант Б: Python + FastAPI
- **За:** проще интеграция с ML/AI библиотеками
- **Против:** другой язык от фронта, сложнее поддерживать маленькой командой

---

### AI / LLM

#### Вариант А: YandexGPT API (выбранный)
- Российский провайдер, хранение данных в РФ
- ФЗ-152 compliant из коробки
- [YandexGPT API документация](https://yandex.cloud/ru/docs/yandexgpt/)
- **За:** нет проблем с санкциями, рублёвые расчёты, персональные данные не покидают РФ
- **Против:** менее мощный, чем GPT-4; меньше сообщество

#### Вариант Б: GigaChat API (Сбер)
- [GigaChat API](https://developers.sber.ru/portal/products/gigachat)
- **За:** сильная языковая модель для русского языка
- **Против:** зависимость от Сбера как конкурента; сложнее конкурировать при партнёрстве

#### Вариант В: OpenAI через прокси (запасной)
- Использование Azure OpenAI через европейские дата-центры
- **За:** GPT-4 — лучшее качество рассуждений
- **Против:** юридические риски, данные покидают РФ, нарушение ФЗ-152

#### Вариант Г: Гибрид YandexGPT + OpenAI (резервный)
- YandexGPT для основного потока; GPT-4 через прокси для сложных запросов
- **За:** баланс качества и compliance
- **Против:** сложная архитектура

---

### База данных

#### Вариант А: PostgreSQL + Supabase self-hosted (выбранный)
- Supabase развёрнут на Yandex Cloud в зоне ru-central1
- Row Level Security для изоляции пользовательских данных
- **За:** быстрый старт, знакомый SQL, RLS, realtime

#### Вариант Б: MongoDB Atlas (запрещён — данные за рубежом)

#### Вариант В: Yandex Managed PostgreSQL
- Полностью управляемый в Yandex Cloud
- **За:** проще для маленькой команды, нет DevOps
- **Против:** vendor lock-in

---

### Хостинг и инфраструктура

#### Вариант А: Yandex Cloud (выбранный)
- Serverless Functions + Managed PostgreSQL + Object Storage
- [Yandex Cloud](https://yandex.cloud/)
- **За:** российские дата-центры, рублёвые расчёты, ФЗ-152

#### Вариант Б: Selectel
- [Selectel](https://selectel.ru/)
- **За:** дешевле, хорошая поддержка
- **Против:** меньше managed-сервисов

---

## Решение

**Принятый стек:**

```
Frontend:  Telegram Mini App (React + Vite + TailwindCSS)
Backend:   Node.js + Fastify + TypeScript
AI/LLM:    YandexGPT API (основной) + GigaChat API (резервный)
Database:  Yandex Managed PostgreSQL
Cache:     Redis (Yandex Managed Redis)
Hosting:   Yandex Cloud (Serverless + MDB)
Charts:    Recharts (встроен в React-компоненты TMA)
Auth:      Telegram initData (нативная авторизация TMA)
```

---

## Последствия

- Все персональные данные хранятся в Yandex Cloud ru-central1 → ФЗ-152 compliant
- Авторизация через Telegram initData — никаких паролей, нулевой friction
- Для более сложных AI-запросов оставляем архитектурную возможность подключить OpenAI через Azure при смягчении регуляторики
